# Personal WIP Limits — User Guide

Personal WIP Limits помогают контролировать загрузку каждого участника команды. Вы задаёте максимальное количество задач, которые человек может иметь одновременно, и доска показывает, кто укладывается в лимит, а кто перегружен.

## Быстрый старт

1. Откройте настройки доски (Board settings), вкладка **Columns**
2. Нажмите кнопку **"Manage per-person WIP-limits"**
3. В открывшемся окне найдите пользователя по имени через поле поиска
4. Укажите лимит задач и нажмите **"Add limit"**
5. Нажмите **"Save"** для сохранения

После сохранения на доске появится аватар пользователя с счётчиком задач.

## Настройка лимита

### Добавление

В модальном окне настроек заполните форму:

- **Person JIRA name** — начните вводить имя, выберите пользователя из выпадающего списка
- **Max issues at work** — максимальное количество задач (минимум 1)

По умолчанию лимит считает все задачи во всех колонках и свимлейнах. Вы можете сузить область подсчёта:

- **Columns** — снимите галочку "All columns" и выберите только нужные колонки
- **Swimlanes** — снимите галочку "All swimlanes" и выберите только нужные свимлейны
- **Issue types** — снимите галочку "Count all issue types", укажите проект и выберите типы задач

#### Примеры

| Что хотите | Как настроить |
|-----------|--------------|
| Макс. 5 задач на всей доске | Limit: 5, всё остальное — All |
| Макс. 3 задачи в "In Progress" | Limit: 3, Columns: In Progress |
| Макс. 2 бага в "Code Review" | Limit: 2, Columns: Code Review, Issue types: Bug |
| Макс. 4 задачи в Frontend свимлейне | Limit: 4, Swimlanes: Frontend |

#### Несколько лимитов для одного человека

Можно создать несколько лимитов с разными фильтрами для одного пользователя. Например:
- Не более 3 задач в "In Progress"
- Не более 2 багов во всех колонках

Каждый лимит считается и отображается отдельно.

### Редактирование

Нажмите **"Edit"** в строке лимита. Форма заполнится текущими значениями, кнопка сменится на **"Edit limit"**. Внесите изменения и нажмите **"Edit limit"** для сохранения, или **"Cancel"** для отмены.

### Удаление

Нажмите **"Delete"** в строке лимита. Лимит удаляется сразу, без подтверждения.

### Сохранение

Все изменения (добавление, редактирование, удаление) применяются только после нажатия кнопки **"Save"** в модальном окне. **"Cancel"** отменяет все несохранённые изменения.

## Отображение на доске

После настройки лимитов в верхней панели доски появляются аватары пользователей.

### Счётчик

Под каждым аватаром отображается счётчик в формате **текущее / лимит**:

| Счётчик | Цвет | Значение |
|---------|------|----------|
| 2 / 5 | Зелёный | В пределах лимита |
| 5 / 5 | Жёлтый | Лимит достигнут |
| 7 / 5 | Красный | Лимит превышен |

При превышении лимита карточки этого пользователя, входящие в подсчёт, подсвечиваются красным фоном.

### Фильтрация

Клик по аватару включает режим фильтрации — на доске остаются только задачи, входящие в подсчёт этого лимита. Остальные карточки скрываются. Повторный клик снимает фильтр и возвращает все карточки.

Если у пользователя несколько лимитов, каждый показывается отдельным аватаром. Клик фильтрует задачи именно по выбранному лимиту.

## Важно знать

- У пользователя должен быть установлен **Jira-аватар**, иначе лимит не будет корректно отображаться
- Настройки лимитов сохраняются в свойствах доски и **видны всем** участникам
- Минимальный лимит — **1** (ноль автоматически исправляется на 1)
- Нельзя создать два одинаковых лимита для одного человека с одинаковым набором фильтров
