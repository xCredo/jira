# Personal WIP Limits

## Проблема

На доске не видно, что человек перегружен задачами — это становится заметно, когда уже поздно.

## Решение

Индивидуальные лимиты на количество задач в работе. Доска показывает загрузку каждого человека и предупреждает о превышении.

## Настройка

1. Открыть настройки доски, вкладка Columns
2. Нажать "Manage per-person WIP-limits"
3. Найти пользователя по имени, задать лимит, нажать "Add limit"
4. Нажать "Save"

Можно ограничить лимит конкретными колонками, свимлейнами или типами задач. Например: не более 3 задач в "In Progress", или не более 2 багов на всей доске.

Для одного человека можно создать несколько лимитов с разными фильтрами.

### Редактирование и удаление

- Кнопка "Edit" в таблице лимитов загружает данные в форму для изменения
- Кнопка "Delete" удаляет лимит
- Кнопка "Cancel" отменяет редактирование без сохранения изменений

### Валидация

- Нельзя добавить лимит без выбора пользователя
- Нельзя создать дубликат — два лимита с одинаковым набором фильтров для одного человека
- Минимальный лимит — 1

## На доске

Аватары людей с лимитами появляются в верхней панели. Под каждым — счётчик:

- **2 / 5** (зелёный) — в пределах лимита
- **5 / 5** (жёлтый) — на грани
- **7 / 5** (красный) — превышен, карточки подсвечены

Клик по аватару оставляет на доске только задачи, входящие в этот лимит. Повторный клик снимает фильтр.

### Что считается

Задача попадает в подсчёт, если она:
- Назначена на этого пользователя
- Находится в одной из выбранных колонок (или в любой, если выбрано "All")
- Находится в одном из выбранных свимлейнов (или в любом, если выбрано "All")
- Имеет один из выбранных типов задач (или любой тип, если выбрано "All")

## Ограничения

- У пользователя должен быть установлен Jira-аватар
- Настройки доступны всем, кто видит доску
