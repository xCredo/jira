# Групповые лимиты WIP по колонкам (CONWIP)

Задаёт один общий лимит work-in-progress сразу на несколько колонок: карточки в этих колонках суммируются, при необходимости с учётом swimlane и типа задачи.

## Что делает

- Объединяет колонки в именованную группу с одним максимумом задач, которые могут одновременно находиться в этой группе.
- По желанию учитывает только задачи из выбранных swimlane или из всех swimlane.
- По желанию учитывает только выбранные типы задач для этой группы.
- Можно задать цвет заголовка колонок группы; если не задавать — цвет подберётся по имени группы.
- На доске видно, насколько группа заполнена относительно лимита, и когда лимит превышен.

## Как настроить

1. Откройте **Board settings** для Scrum- или Kanban-доски (нужны правки конфигурации доски).
2. Перейдите на вкладку **Columns**.
3. Нажмите **Column group WIP limits** (точная подпись может совпадать с языком Jira).
4. В диалоге перетащите колонки в группы. Для каждой группы задайте лимит и при необходимости цвет, типы задач и какие swimlane входят в счётчик.
5. Нажмите **Save**, чтобы сохранить настройки для всех на этой доске, или **Cancel**, чтобы закрыть без сохранения.

Сохранять может только тот, кому в Jira разрешено менять настройки доски (обычно администратор доски).

## Поведение на доске

- На первой колонке группы показан индикатор в духе **текущее / лимит**, чтобы сразу видеть загрузку.
- Заголовки колонок оформлены так, чтобы группу было легко узнать.
- При превышении лимита затронутые области колонок подсвечиваются **красным**; что именно считается, по-прежнему определяется вашими правилами по swimlane.
- Подсчёт задач согласован с настройками доски в Jira (в том числе учёт или исключение подзадач), как в обычном поведении доски.

# Персональные лимиты WIP

Ограничивает, сколько задач у каждого участника может быть «в работе» на доске, с наглядным статусом и возможностью сфокусироваться только на задачах, входящих в его лимит.

## Что делает

- Можно задать несколько лимитов на одного человека — у каждого свой потолок и своя область действия.
- **Область** — какие колонки и swimlane учитываются (и при необходимости какие типы задач). Если ничего не выбрано, обычно подразумевается вся доска для этого лимита.
- На доске у каждого настроенного участника есть компактный индикатор рядом с заголовком доски.

## Как настроить

1. Откройте **Board settings** доски.
2. Перейдите на вкладку **Columns**.
3. Нажмите **Manage per-person WIP-limits**.
4. В окне найдите пользователя, задайте максимум задач и при необходимости сузьте колонки, swimlane и типы задач. Добавляйте, правьте или удаляйте строки.
5. Нажмите **Save**, чтобы применить для всех на доске, или **Cancel**, чтобы отменить изменения.

Сохранять могут только те, кому разрешено менять конфигурацию доски.

## Поведение на доске

- **Бейджи с аватарами** показывают загрузку: зелёный ниже лимита, жёлтый на лимите, красный выше.
- Задачи, которые входят в лимит и выводят человека **за** порог, могут получить **красную подсветку** карточки (если для этого лимита это включено).
- **Клик по бейджу** включает или выключает фильтр: остаются только задачи, попадающие в область этого лимита — удобно на стендапе или при разборе перегруза.
- Когда фильтр включён, несоответствующие части доски могут скрываться, пока снова не нажмёте на бейдж.

# Лимиты WIP по ячейкам доски

Задаёт один общий лимит work-in-progress на выбранный блок ячеек доски (пересечение колонки и swimlane), с контуром диапазона, опциональными счётчиками на ячейках и явным оформлением при превышении.

## Что делает

- Можно задать **диапазон** ячеек и один общий лимит на все задачи, которые находятся внутри этих ячеек.
- Вы сами выбираете, какие пересечения колонка–swimlane входят в диапазон; для каждой ячейки можно включить или выключить числовой индикатор.
- По желанию учитываются только выбранные типы задач.
- Диапазон можно пометить как **визуально отключённый** — показывается штриховка «выкл.», без активного поведения бейджа лимита.
- Задачи во всех ячейках диапазона суммируются в **один** счётчик относительно этого лимита.

## Как настроить

1. Откройте **Board settings**.
2. Перейдите на вкладку **Columns**.
3. Нажмите **Edit WIP limits by cells**.
4. Добавляйте, редактируйте или удаляйте диапазоны. Для каждого укажите имя, лимит, нужно ли отображать как отключённый, при необходимости типы задач и таблицу пар swimlane/колонка; для ячейки отдельно включите или выключите индикатор.
5. Нажмите **Save**, чтобы сохранить для доски, или **Cancel**, чтобы выйти без сохранения.

Сохранённые диапазоны действуют для всех, кто смотрит доску с расширением.

## Поведение на доске

- Закрытый блок ячеек обведён **синей пунктирной** рамкой, чтобы форма диапазона была заметна.
- На ячейках с включённым индикатором бейдж показывает **сколько задач** в диапазоне относительно **лимита**, цвет отражает «ниже / на / выше лимита».
- При **превышении** лимита ячейки диапазона получают **полупрозрачный красный** фон.
- **Отключённые** диапазоны показывают штриховку поверх — видно, что область намеренно не используется как активный лимит.

Подсчёт задач согласуется с настройками доски (в том числе учёт подзадач), как и в других WIP-функциях расширения.

# Лимиты WIP по swimlane

Задаёт отдельный лимит work-in-progress для каждой строки swimlane и при необходимости правила по типам задач; индикатор на заголовке дорожки — удобно для полос вроде «Expedite» с особыми правилами WIP.

## Что делает

- Для каждой swimlane можно задать числовой лимит или оставить дорожку без лимита.
- По желанию учитываются только выбранные типы задач в этой дорожке.
- Дополняет **групповые лимиты по колонкам**: общие правила по колонкам можно сочетать с лимитами на отдельных строках.
- Завершённые задачи в эти лимиты не входят; подзадачи учитываются так же, как в других WIP-функциях расширения на этой доске.

## Как настроить

1. Откройте **Board settings**.
2. Перейдите на вкладку **Swimlanes**.
3. В стандартном элементе Jira выберите стратегию swimlane **Custom**. Для этой функции нужна стратегия **Custom**.
4. Нажмите **Configure WIP Limits**.
5. В модальном окне для каждой swimlane доски есть строка: укажите лимит и при необходимости типы задач. **OK** сохраняет все изменения, **Cancel** закрывает без сохранения.

Сохранять может администратор доски или пользователь с правом менять конфигурацию доски.

## Поведение на доске

- Если у swimlane задан лимит, в заголовке строки показывается бейдж **счётчик / лимит**.
- При **превышении** лимита заголовок **подсвечивается**, чтобы перегруз было видно сразу.
- Подсчёт следует сохранённым правилам по дорожке и типам; у полос без активного лимита поведение как у обычных swimlane в Jira (если не включены другие WIP-функции).

# Лимиты WIP по значениям полей

Превращает поле на карточке доски в лимит в духе «ёмкости»: можно считать или суммировать вклад задач по значению поля (или по факту заполнения поля), ограничить область колонками и swimlane, показать бейджи рядом с доской и подсветить карточки при перегрузе.

## Что делает

- Каждое правило выбирает **поле** из **раскладки карточки** доски — учитываются только поля, которые реально отображаются на карточке.
- Задаётся способ учёта: например **если поле заполнено**, **ровно одно значение**, **любое из нескольких значений** или **сумма чисел** по подходящим карточкам — в окне настройки режимы должны быть понятны по подписям.
- Для правила задают **короткое имя** для бейджа, числовой **лимит**, по желанию **цвет фона бейджа**, а также при необходимости фильтры по **колонке** и **swimlane**, чтобы правило действовало только где нужно.
- На бейдже видно **текущее значение и лимит**, цветами отмечены «ниже / на / выше лимита».

## Как настроить

1. Откройте **Board settings**.
2. Перейдите на вкладку **Card layout** (нужные поля должны быть на карточке).
3. Нажмите **Edit WIP limits by field**.
4. Добавляйте, правьте или удаляйте лимиты: поле, режим расчёта, значения (если нужны), отображаемое имя, лимит и при необходимости область по колонке/swimlane и цвет бейджа.
5. Сохраните в модальном окне — настройка хранится на доске для всех, кто пользуется расширением.

Сохранять может только тот, кому разрешено менять конфигурацию доски.

## Поведение на доске

- Над колонками доски, в зоне панели с фильтрами и переключателями вида, появляется ряд **бейджей** — по одному на каждый настроенный лимит — с вашими подписями и цветами.
- При **превышении** лимита карточки, которые **входят** в этот счётчик, получают общую **красноватую подсветку**, чтобы перегруженные корзины были заметны.
- Счёт обновляется при перемещении карточек, с учётом настроек статистики доски Jira (например с исключением подзадач, если так задано на доске).

# Цвета карточек

Делает правила **Card Colors** в Jira заметнее: по желанию заливает всю карточку задачи мягким оттенком того же цвета, а не только узкую полоску слева.

## Что делает

- Использует те же JQL-правила цветов карточек, что вы уже настроили в Jira; меняется только сила проявления цвета (вся карточка или только полоска).
- Добавляет светлый, читаемый фоновый оттенок в тон назначенному карточке цвету.
- Не меняет оформление **помеченных** (flagged) карточек, чтобы сигнал срочности оставался на виду.
- Не меняет карточки, у которых уже есть отдельная подсветка (например предупреждения лимитов WIP), чтобы предупреждения оставались заметными.

## Как настроить

1. Откройте доску.
2. Перейдите в **Board settings** (шестерёнка на доске).
3. Откройте вкладку **Card Colors** (стандартный экран Jira, где задаются цвета через JQL).
4. Над таблицей правил найдите опцию **Fill whole card** (в подсказке — краткое объяснение «до и после»).

Включать и выключать могут те, кто может менять настройки доски — обычно **администраторы доски**. Выбор сохраняется вместе с остальными настройками доски.

## Поведение на доске

Если **Fill whole card** включён, у карточек на доске может появляться мягкая заливка по всей карточке в соответствии с правилами Card Colors. Полоска слева от Jira остаётся; остальная область карточки получает согласованный светлый фон. Помеченные карточки и карточки с отдельными подсветками сохраняют привычный вид. Если опция выключена, карточки выглядят как в обычном Jira (только полоска).

# Гистограмма по swimlane

Показывает небольшую **столбчатую диаграмму** рядом с названием каждого swimlane, чтобы сразу видеть, как задачи в этом swimlane **распределены по колонкам** доски.

## Что делает

- Для каждого swimlane считает, сколько задач в каждой колонке, и рисует один компактный график в заголовке swimlane.
- Высота столбца отражает долю задач этой колонки среди всех задач swimlane (самый высокий столбец — наибольшая доля).
- При наведении (или фокусе) на график показываются имя колонки и число задач.
- Нейтральные серые столбцы: более светлые «ячейки» для пустых колонок, темнее там, где есть задачи.
- Обновляется автоматически при изменениях на доске.

## Как настроить

Ничего настраивать не нужно. Диаграмма появляется сама на поддерживаемых видах доски, когда включены swimlanes.

## Поведение на доске

Рядом с названием каждого swimlane отображается горизонтальный ряд узких столбцов по числу колонок доски. Мини-график отражает только задачи внутри этого swimlane. На некоторых **упрощённых или альтернативных видах** доски диаграмма может не показываться; откройте привычный детальный вид Scrum/Kanban-доски, чтобы её увидеть.

# Размытие данных

**Режим приватности и презентаций:** делает текст на страницах Jira нечитаемым, а многие изображения — размытыми, чтобы можно было показать экран или провести демо без раскрытия заголовков задач, ключей, имён и описаний; вёрстка остаётся на месте, содержимое не прочитать.

## Что делает

- Сильно «смазывает» текст и размывает подходящие изображения по всему Jira — не только на досках, но и в карточке задачи, поиске, бэклоге и части экранов настроек.
- Запоминает ваш выбор: после включения новые вкладки и перезагрузки страниц сохраняют размытие, пока вы его не отключите.
- Переключается из меню браузерного расширения, без захода в **Board settings** в Jira.

## Как настроить

1. На странице Jira, где активно расширение, откройте **контекстное меню** страницы (правый клик по странице или меню вкладки в браузере).
2. Найдите пункт расширения **Blur secret data** и используйте его как **чекбокс**: отмечено — размытие включено, снято — выключено.

Отдельного переключателя в **настройках доски** в Jira нет. Настройка сохраняется между переходами по страницам.

## Поведение на доске

При включённом размытии на доске и в бэклоге вместо читаемых названий и полей виден «смазанный» текст; детали не прочитать, а колонки и расположение карточек остаются понятными. После презентации или шаринга экрана отключите размытие тем же пунктом меню.

# Additional Card Elements

Добавляет на карточки доски и бэклога дополнительные подписи и статусы: связи, «старение» в колонке, дедлайны и наглядные сигналы по вашим правилам.

## Что делает

### Отображение связей задач (Issue Links Display)

- Показывает связанные задачи компактными бейджами под summary карточки.
- Можно задать несколько правил (тип связи, направление, при необходимости фильтры), чтобы отображались только нужные связи.
- Доступны цвета и многострочное отображение для связанных задач.

### Дни в колонке (Days in Column)

- Показывает, сколько дней задача находится в текущей колонке, в виде бейджа на карточке.
- Пороги «внимания» задаются глобально или по колонкам (например, стиль предупреждения после выбранного числа дней).

### Дни до дедлайна (Days to Deadline)

- Показывает бейдж по выбранному вами полю даты или даты-времени на задаче.
- Режимы: всегда показывать; только если до даты осталось не больше N дней или срок уже прошёл; только просрочка.
- При необходимости — стиль предупреждения до наступления дедлайна.

### Проверки условий (Condition Checks)

- Показывает маленькие иконки на карточке, когда задача (или связанная с ней работа по правилу) попадает под заданное условие.
- В правило при необходимости можно включить подзадачи, дочерние задачи эпика или связанные задачи — в зависимости от настройки.
- У каждого правила могут быть своя иконка, текст подсказки (tooltip), цвет и при необходимости анимация.

**Прочее поведение**

- В виде бэклога на карточках показываются только бейджи связей.

## Как настроить

1. Откройте доску Jira, затем настройки **Jira Helper** для этой доски.
2. Откройте вкладку **Additional Card Elements**.
3. Включите фичу главным переключателем.
4. Укажите **колонки**, в которых на карточках должны отображаться дополнительные бейджи.
5. При необходимости включите **Show links in backlog**, чтобы бейджи связей на карточках отображались и в бэклоге.
6. Настройте блоки связей, дней в колонке, поля дедлайна и режимы, а также правила условий в соответствующих секциях. **Reset all settings** возвращает настройки этой вкладки к значениям по умолчанию.

Настройки действуют для конкретной доски. Их могут менять администраторы доски.

## Поведение на доске

- В **выбранных колонках** на карточках отображаются настроенные бейджи: связи под summary; дни в колонке и/или дедлайн ближе к краю карточки; иконки условий — в зоне внизу карточки.
- Если включено **Show links in backlog**, в **бэклоге** у карточек показывается горизонтальная полоса бейджей связей; остальные типы бейджей остаются на основной доске.

# Sub-tasks progress

Показывает на карточках доски составной прогресс (полосы и/или компактные счётчики) по дочерней и связанной работе — подзадачам, задачам под эпиком, связям и при необходимости внешним ссылкам — в разрезе To Do, In Progress, Done и Blocked.

## Что делает

- Делит работу на **четыре категории**: to do, in progress, done и blocked — по тому, как вы относите **статусы** Jira к этим категориям (часть статусов можно **не учитывать** в сумме).
- Позволяет выбрать, **что считать**: например дети эпика, подзадачи, связанные задачи (с опциональным ограничением по **типу связи**) и при необходимости «внешние» элементы — каждый источник включается отдельно.
- Поддерживает **группировку** — например по проекту, исполнителю или типу задачи — или **свои** группы по полю или фильтру, с отдельным оформлением для полос и простых счётчиков.
- Может показывать **многосегментные** полосы, **одну** сводную полосу и/или **числовые счётчики**; группы можно скрывать, когда всё выполнено, или показывать только незавершённое.
- По желанию учитывает **флаги** Jira и связи вида «blocked by» как blocked в подсчётах и может показывать на карточке **краткие предупреждения**, если что-то заблокировано или помечено флагом.
- **Цвета** — из готовой схемы или своей палитры для сегментов.

## Как настроить

1. Откройте доску Jira и перейдите в настройки **Jira Helper** для этой доски.
2. Откройте вкладку **Sub-tasks progress**.
3. Включите фичу. **Reset all settings** возвращает настройки этой вкладки к значениям по умолчанию.
4. В блоке выбора **колонок** укажите, **в каких колонках показывать прогресс** на карточках.
5. В секции подсчёта включите нужные **источники** (дети эпика, подзадачи, связи, внешние ссылки и т.д.) и при необходимости фильтры по типам связей.
6. Настройте **группировку**: поле по умолчанию, игнорируемые группы и пользовательские группы (имена, правило группы, полоса или счётчик, цвета, когда скрывать или показывать).
7. Сопоставьте **статусы** с четырьмя категориями прогресса; при необходимости задайте **цвета** в своей схеме.
8. При необходимости включите обработку **blocked / flagged** и связанные предупреждения.

Настройки задаются **на доску** в Jira Helper. Сохранять их могут те, у кого есть право менять настройки Jira Helper для этой доски.

## Поведение на доске

- В **отслеживаемых колонках** на каждой карточке отображаются настроенные полосы и/или счётчики с выбранной группировкой.
- Учёт blocked и flagged появляется только если вы это включили; текст предупреждения на карточке показывается, когда срабатывают правила.

# Issue view helpers

Помогает заметить задачи с **флагом** в иерархии на экране задачи и даёт быстрый способ **скрыть или показать правую панель** в классическом полноэкранном виде задачи.

## Что делает

- Подсвечивает строки **иерархии задачи** (связанные задачи, подзадачи, дочерние задачи эпика), если эти задачи **помечены флагом** в Jira, и показывает рядом **иконку флага**.
- Для **открытой сейчас** задачи, если у неё стоит флаг, дополнительно показывает флаг рядом с типичным блоком в шапке (priority / type) в **classic** issue view.
- Добавляет компактный переключатель **свернуть / развернуть** для **правой боковой панели** на legacy-экране задачи, чтобы освободить место по ширине и при необходимости снова открыть панель.

## Как настроить

Ничего настраивать не нужно. Помощники работают автоматически на страницах задач и в боковой панели задачи на доске. Отдельные настройки доски или расширения не требуются.

## Поведение на доске

- Если с доски открыта задача в **боковой панели деталей**, связанные элементы иерархии с флагом в этой панели подсвечиваются и помечаются иконкой так же, как на полной странице (оформление может немного отличаться в **classic** и **new** issue view).
- Переключатель **правой панели** показывается на **полной странице задачи** и при просмотре задачи из **поиска задач**.
- Переключатель боковой панели доступен в **classic** (классическом) виде задачи.

# Дополнения к Control Chart

Добавляет на отчёт Jira **Control Chart** визуальные подсказки: опорную линию SLA с быстрым предпросмотром и по желанию сетку измерений для считывания lead/cycle time относительно шагов в духе story points.

## Что делает

- Показывает на диаграмме горизонтальную линию **SLA** с полупрозрачной полосой, подписями с числом дней и долей задач на уровне SLA или ниже, а также пункт **SLA** в легенде диаграммы.
- Линия обновляется по мере изменения значения SLA.
- Позволяет редакторам доски **сохранить** значение SLA для доски, чтобы оно подставлялось при следующих визитах.
- Добавляет опциональную **сетку измерений** в опциях диаграммы: пресеты **Fibonacci** или **linear**, перетаскиваемый и масштабируемый оверлей для выравнивания горизонтальных линий с диаграммой.

## Как настроить

1. Откройте **Reports** по доске и выберите отчёт **Control Chart**.
2. В блоке опций диаграммы задайте **SLA** в **days** — диаграмма обновится сразу.
3. Чтобы **сохранить** SLA для всех, кто пользуется этой доской, нажмите **Save** (если у вас есть право редактировать доску).
4. Для сетки: в опциях диаграммы включите её чекбоксом, выберите пресет **Fibonacci** или **linear**, при необходимости сдвиньте и измените размер оверлея на диаграмме.

## Поведение на доске / на странице

На **Control Chart** отображается стандартная диаграмма Jira плюс элементы управления SLA из расширения в колонке опций. При заданном SLA видны линия, полоса и подписи в легенде. При включённой сетке поверх диаграммы показываются горизонтальные направляющие для оценки дней и шага. Сохранённый SLA появится при следующем открытии Control Chart этой доски.

# Шаблон описания (создание задачи)

Ускоряет заполнение поля **Description** при создании задачи или подзадачи: можно вставить готовую структуру и сохранить свой текст как шаблон на следующий раз.

## Что делает

- Добавляет рядом с полем описания в диалоге создания кнопки **Add template** и **Save template** (иконки пера и сохранения).
- **Add template** вставляет в описание структурированный «баг-шаблон». Есть встроенный шаблон по умолчанию; можно сохранить свой вместо него.
- **Save template** сохраняет текущее описание как ваш личный шаблон.
- Ваш шаблон хранится локально в этом браузере.

## Как настроить

Отдельного экрана настроек нет. Откройте создание задачи или подзадачи как обычно.

- Нажмите **Add template**, чтобы вставить шаблон в поле описания.
- Отредактируйте текст и при необходимости нажмите **Save template**, чтобы текущий plain-text описания стал новым шаблоном.

Права Jira на это не влияют: шаблон только на вашем устройстве в этом профиле браузера.

## Поведение на доске / на странице

Когда открыт диалог **create issue** или **create subtask** и видна область описания, рядом появляются две кнопки. После закрытия и повторного открытия диалога они снова доступны так же. Остальной интерфейс Jira (доски, бэклог, настройки) не меняется.

# Панель настроек доски

Открывает с agile-доски одно окно **Jira Helper**, чтобы настроить параметры расширения, которые действуют, пока вы работаете с доской.

## Что делает

- Показывает рядом с сайдбаром доски элемент **Jira Helper**; по клику открывается всплывающее окно с **вкладками**.
- Каждая вкладка — отдельная группа настроек из расширения (например язык интерфейса, диагностика, элементы карточек).

## Как настроить

1. Перейдите на **Scrum**- или **Kanban**-**board** (обычный вид agile-доски).
2. Найдите элемент **Jira Helper** у левого сайдбара и нажмите его.
3. Переключайте **вкладки** вверху окна и меняйте параметры внутри каждой вкладки по её назначению.

## Поведение на доске / на странице

На **доске** элемент **Jira Helper** остаётся доступным при работе с колонками и карточками. Окно настроек показывается поверх доски; закрытие возвращает вас к доске.

# Локальные настройки

Задаёт **язык**, на котором расширение Jira Helper показывает свои подписи и сообщения в вашем браузере.

## Что делает

- Добавляет вкладку **Local Settings** в окно настроек с доски (через **Jira Helper** рядом сайдбаром).
- Даёт выбор **UI language**: как в браузере (**Auto**), **English** или **Russian**.
- Запоминает выбор в этом браузере и **не** меняет язык самой Jira.

## Как настроить

1. Откройте **board**, нажмите **Jira Helper** у сайдбара и перейдите на вкладку **Local Settings**.
2. В **UI language** выберите **Auto**, **English** или **Russian**.

Изменение применяется сразу после выбора, отдельной кнопки «Сохранить» нет. Настройка личная и сохраняется локально в вашем браузере.

## Поведение на доске / на странице

После выбора языка интерфейс расширения (всплывающее окно настроек доски и подписи функций) отображается на **English**, **Russian** или как в системе браузера при **Auto**. Меню и язык Jira не меняются.
